==========================
Pretrained neural vocoders
==========================

This is the project folder pre-trained neural vocoders.
Demonstration scripts are provided to show how to do copy-synthesis on a toy dataset
usign the pre-trained neural vocoders.

A few models are available: HiFiGAN, HN-sinc-NSF, WaveGlow, NSF with HiFiGAN.

Different versions are available:
1. pretrained on VoxCeleb2 dev
2. pretrained on ASVspoof 2019 training set bona fide
3. pretrained on LibriTTS subset 
4. pretrained on LibriTTS subset then fine-tuned ASVspoof 2019 training set bona fide
5. pretrained on MLS English subset 

Versions 2-4 correspond to voc.v2, voc.v3, voc.v4 in paper
Xin Wang, and Junichi Yamagishi. Spoofed training data for speech spoofing countermeasure 
can be efficiently created using neural vocoders. Proc. ICASSP, 2023 (to appear)

If you want to train the neural vocoders, please check the last part of README.

----------------
Folder structure
----------------
 
Folders with "=" are those to be generated after running the scripts
|
|- code: code and scripts to extract acoustic features
|   |- 00_setup.sh:              script to setup environment                
|   |- 01_feature_extraction.sh: script to acoustic feature from natural waveforms
|   |- 02_vocoding.sh:           script to do copy-synthesis given features from 01
|   |
|   |- try_*.sh:                 example scripts for sanity check
|   |
|   |- feature_extraction.py: python code called by 01_feature_extraction.sh
|
|- data: data for demonstration (16kHz waveform, PCM format)
|   |- voxceleb2: voxceleb samples (anonymized for privacy protection)
|   |              To protect the privacy of the speakers in VoxCeleb2, 
|   |              audio samples here have been anonymized using DSP tools
|   | 
|   |- asvspoof19_bona: samples from ASVspoof 2019 LA bona fide data
|   
|- pretrained_voxceleb2: pretrained models using voxceleb2 dev set
|   |- hifigan:        folder for pre-trained model and code of hifigan
|   |    |- config.py  configuration file to setup feature dimensions and so on
|   |    |- model.py   model definition
|   |    |- main.py    main function
|   | 
|   |- hn_sinc_nsf: ...  for hn_sinc_nsf
|   |- hn_sinc_nsf_hifi: ... for hn_sinc_nsf_hifi
|   |- waveglow:  ... for waveglow
|
|- pretrained_asvspoof_trn_bona: pretrained models using ASVspoof2019 trn bona fide
|   |- ...
|
|- pretrained_libritts_subset:   pretrained on LibriTTS subset
|- pretrained_libritts_asvspoof_trn_bona: pretrained on LibriTTS subset, then fine-tuned 
|
| 
|= data-feat: folder to store the extracted acoustic features
|   |= asvspoof19_bona: extracted features for asvspoof19_bona
|   |    |              structure of this folder will be mirrored from that in data
|   |    |= file.lst:   file list generated by 01_feature_extraction.sh
|   |    |              it will be used by 02_*.sh
|   |    |= trn:        name of the subfolder, 
|   |        |= *.mel:  Mel-spectrogram
|   |        |= *.f0:   F0  
|   |
|   |= voxceleb2:       extracted features for voxceleb2
|   
|= data-vocoded:
|   |= pretrained_voxceleb2: 
|         |= hifigan
|              |= asvspoof19_bona: vocoded waveforms of asvspoof19_bona, using hifigan
|              |                   in pretrained_voxceleb2
|              |= voxcelb2:        ...
|   ...


--------------------
Usage: demonstration
--------------------
Let's use the sample data in ./data for demonstration

# go do code
cd code

# download scripts and install dependency (for the 1st time)
bash 00_setup.sh

# extract acoustic features on the provided data saple
bash 01_feature_extraction.sh

# do vocoding using model specified in the script (hifigan)
bash 02_vocoding.sh


--------------------
Usage: vocoding on full voxceleb2 
(or a large dataset)
--------------------
Apologize that you have to do some manual work

1. please prepare the voxceleb2 data in WAV format (.wav PCM)
   (You may also change feature_extraction.py to directly load aac)
   
2. please follow the example above, modify 01 and 02 scripts if necessary
   and run the scripts.

3. running the code requires a GPU. Models except WaveGlow will take a few hours.
   WaveGlow will take a few days I think.
